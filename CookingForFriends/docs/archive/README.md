# Archive

These files are kept for traceability. They may describe older designs,
one-off implementation prompts, migration history, or superseded incident logs.

Do not start implementation from archived docs unless a current top-level doc
explicitly points here for historical context.

Categories:

- `architecture/` — older architecture snapshots
- `incidents/` — legacy bug reports and divergent incident logs
- `materials/` — dated experiment-material notes and generated content drafts
- `migrations/` — migration reports
- `plans/` — old implementation plans and prompt-style specs
- `reviews/` — dated codebase or architecture reviews
