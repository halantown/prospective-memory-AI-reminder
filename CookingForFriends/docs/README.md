## Directory Rules

- `development/`: implementation docs that should match current code.
- `operations/`: incident logs, runbooks, and operational records.
- `research/`: study materials, storyboards, scripts, and assets.
Keep this README small. Add details to the relevant supporting document instead
of creating more root-level docs.
